3D rasterization

3d rasterization

My 3d rasterization engine was a Java project built for CS 1332 - the intro to data structures and algorithm class at Georgia Tech. I built the project in 2 weeks, using only the core java libraries. I implemented a software rasterization algorithm that takes a triangle defined by three colored input points and then draws it on a given raster image (producing a z-buffer for proper culling in the process). I then built 3d shape models and the code to scale, rotate, and translate them in the scene.

selfieaware.com

selfieaware

SelfieAware gives people selfiness scores based on the number of selfies on their Instagram feed. It was built using facial recognition software and the Instagram API by me and three friends during LAHacks. To use SelfieAware, all you have to do is login through Instagram, and then SelfieAware will automatically browse your recent Instagram pictures and give you a selfiness score based on the ratio of pictures of yourself vs pictures of nature/others.

fabric simulation

fabric simulation

This fabric simulation was a personal project built in fall 2013. It was built in Java using the core Java libraries. To simulate the fabric the program builds an interconnected point mass system, and then simulates the movement of each point mass in discrete timesteps. The final result is interactive - the user's mouse modifies the velocity of nearby point-masses.

queueup.herokuapp.com

queueup

Queueup [q-up] is a party playlist maker. Built at MHacks in 36 hours by me and two friends, Queueup lets you add songs to a central Spotify playlist. If you throw an event, all you have to do is setup a new playlist in the Spotify client app, and then attendees can then simply enter the playlist code on the website to add music and upvote it. Go to the website and type in the code MHACKS to see the playlist we made at MHacks. Add music to it and it will show up in my Spotify!

particle pathfinding

particle pathfinding

This was a personal project in which I developed a pathfinding algorithm for particles. It uses my own software rendering algorithm for the particles alongside a wavefront expansion algorithm I developed for the pathfinding, both implemented in Java. The pathfinding algorithm was optimized to work with many pathfinders, with over 20k particles pathfinding to the goal node in this screenshot. Check out the accompanying article to better understand the algorithm.

particle rendering

particle rendering

This was a personal project I built as a senior in high school that demonstrates normal mapping and bit hacking for use in software rendering. As you can see, the blocks in the scene are normal mapped, with directional light affecting them appropriately. It's also important to note that I use hacky per-particle lighting, and I built both the particle system and rendering in Java. The final product can handle hundreds of thousands of light-emitting particles.
